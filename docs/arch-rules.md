# Правила Архитектуры И Сопровождения

## Общие Правила

- Проект сопровождается как один репозиторий `support-bot-main`.
- Стараться максимально упрощать изменения и не добавлять лишние компоненты.
- Запускать, тестировать и диагностировать только из корня проекта.
- Основной файл запуска: `docker-compose.yml`.
- Основная документация: `README.md`, `AGENTS.md`, `docs/architecture.md`, `docs/arch-rules.md`.
- Не возвращать отдельные `docs/`, `README.md`, `AGENTS.md`, `.env.example`, `.gitignore` в подпроекты.
- Не хранить секреты в git: `.env`, Telegram token, OpenAI key, Google service account.
- Комментарии в коде писать на русском языке.
- Всю проектную переписку и документацию сопровождения вести на русском языке, если пользователь не попросил иначе.
- Для актуальной документации по библиотекам сначала использовать MCP Context7, затем интернет.

## Docker

- Каждый runtime-компонент должен быть включён в корневой `docker-compose.yml`.
- Не использовать локальное Python-окружение как основной способ запуска.
- Сервис `bot` не публикует порты.
- Сервис `vector-base` публикует только `8080`.
- Сервис `chroma` не публикует порт наружу и доступен только как `chroma:8000` внутри Docker-сети.
- Взаимодействие между сервисами должно идти по Docker DNS, например `http://vector-base:8080`, а не по LAN IP компьютера.

## Документация

- При изменении общей схемы запуска обновлять `README.md` и `docs/architecture.md`.
- При изменении поведения бота обновлять раздел `bot` в `docs/architecture.md`.
- При изменении поиска, синхронизации или ChromaDB обновлять раздел `vector-base` в `docs/architecture.md`.
- Временные документы планирования не нужны: `plan.md` и `requirements.md` не создавать.

## Зависимости

- `support-bot/requirements.txt` и `vector-base/requirements.txt` оставлять в подпроектах: Dockerfile используют их для сборки.
- После изменения зависимостей пересобирать контейнеры и запускать тесты.

## Проверки

После изменений runtime-кода:

```bash
make test
make health
```

После изменений Docker/compose:

```bash
docker compose config --quiet
make up
make ps
make health
```
