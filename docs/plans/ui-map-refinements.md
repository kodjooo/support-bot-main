## (устар.) Сортировки — не требуются

Сортируемые колонки (из кода) — продажи: quantity, refunds, salesAmount,
refundsAmount, deduction, advertisement, profit, profitPerItem, margin, roi, drr,
redemption; заказы — аналогично. Клик по `th.sortable-header` нужной колонки,
активная сортировка = класс `reorder_outer` + `button.reorder_button.active`.

«Все сортировки» = ~12 колонок × (продажи+заказы) × 2 МП ≈ **до 48 почти
одинаковых кадров** (меняется только порядок строк и стрелка). Варианты:
1. **Текстовый факт + 1 пример (рекомендую):** один кадр с активной сортировкой
   как образец + факт «таблицу можно сортировать по колонкам: …» (список из кода).
   Дёшево, полно по смыслу, не засоряет корпус почти-дублями.
2. **Полный перебор:** параметрический генератор в crawl_map (новое поле
   `sort_columns: [...]`), кравлер делает кадр на каждую колонку. Дорого по
   объёму и распознаванию.

→ **Нужно решение пользователя** (по умолчанию возьму вариант 1).
