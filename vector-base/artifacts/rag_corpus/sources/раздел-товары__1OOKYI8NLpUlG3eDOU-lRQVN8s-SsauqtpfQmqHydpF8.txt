Раздел «Товары»
В разделе «Товары» перечислены все артикулы, которые есть в магазине продавца.
Раздел представляет собой таблицу, содержащую:
список товаров;
поле для внесения себестоимости;
тип себестоимости;
поле для комментария.
При необходимости пользователь может вручную указать любой текст в комментарии.
Также в правой части таблицы расположена колонка «Скрыть», где можно выбрать товары, которые необходимо скрыть и не отображать в разделе. При этом такие товары также не будут отображаться в разделе «Склад», однако продолжат учитываться при расчётах прибыли в разделе «Дэшборд».
Фильтрация товаров
В верхней части раздела над таблицей находится фильтр по товарам, в котором можно:
выбрать все товары из ассортимента;
выбрать только определённые товары.
Для поиска можно использовать:
наименование товара;
баркод;
артикул.
Также в правом верхнем углу есть функция «Без себестоимости». При активации с помощью галочки все товары будут отфильтрованы таким образом, что на странице останутся только товары, по которым не внесена себестоимость.
Рядом расположена функция «Показать скрытые». При её активации останутся только товары, которые:
пользователь скрыл вручную;
были скрыты автоматически.
Особенности для Wildberries
На данный момент для магазинов Wildberries доступна возможность получения данных обо всех товарах, включая товары, которые пользователь:
переместил в корзину;
удалил в кабинете продавца Wildberries.
При этом товары, полученные по API корзины, то есть товары, удалённые пользователем в кабинете продавца маркетплейса, автоматически скрываются в разделе «Товары» сервиса дэшборд.
Данные о товарах
В таблице отображаются:
титульная картинка;
наименование товара;
артикул.
Артикул является кликабельным. При клике откроется новая вкладка браузера с карточкой товара на маркетплейсе.
Также для товара отображаются:
баркод;
артикул продавца;
бренд.
Если у товара несколько баркодов, они отображаются как дочерние элементы в аккордионе. Чтобы посмотреть все баркоды, входящие в родительский артикул, необходимо раскрыть аккордеон с помощью кнопки слева от изображения артикула.
Некорректные или автоматически сгенерированные баркоды
Если пользователь видит баркод, который не был предусмотрен пользователем, либо Wildberries автоматически сгенерировал его при создании карточки товара, и сервис по какой-либо причине не смог автоматически объединить его с корректным баркодом, то рядом с таким баркодом будет отображаться информационная иконка в виде восклицательного знака в треугольнике.
При наведении на иконку появится тултип с пояснением, что данный баркод был автоматически сгенерирован Wildberries. Причина появления таких баркодов неизвестна. Wildberries периодически генерирует новые баркоды и передаёт отдельные транзакции с этими баркодами. Для корректного учёта прибыли необходимо вручную объединить такой новый баркод с одним из корректных баркодов данного артикула.
Для этого рядом с уведомлением будет кнопка «Объединить». При клике пользователь сможет выбрать баркод, к которому необходимо привязать новый автоматически сгенерированный Wildberries баркод.
Внесение себестоимости
В разделе «Товары» можно и необходимо вносить себестоимость товаров.
Для внесения себестоимости пользователь может:
кликнуть на ячейку «Себестоимость» непосредственно в таблице;
внести сумму без перехода в другие окна;
кликнуть на поле «Тип» и выбрать вариант «По периодам».
В дальнейшем при клике на ячейку «Себестоимость» также может открываться отдельное окно для внесения себестоимости.
Типы себестоимости
Себестоимость можно внести двумя способами:
1. За всё время
Пользователь указывает одно значение себестоимости, которое будет использоваться:
для всех продаж;
для всех возвратов;
для всех исторических данных;
для текущих данных;
для всех последующих данных.
То есть одно значение применяется ко всему периоду работы сервиса.
2. По периодам
При выборе типа «По периодам» пользователь указывает дату поступления товара.
Дата может быть:
любой исторической датой;
датой в будущем.
Если дата выбрана в прошлом, исторические данные пользователя будут пересчитаны с учётом изменения себестоимости начиная с указанной даты.
После нажатия кнопки «Добавить» появляются две строки:
период «от бесконечности до даты, предшествующей указанной пользователем». В поле «Себестоимость» необходимо указать стоимость, действовавшую до изменения.
период «от указанной пользователем даты до бесконечности».
Все продажи, возвраты и самовыкупы начиная с этой даты будут рассчитываться по новой себестоимости.
Дополнительно можно:
добавить несколько дат поступления;
скрыть товар прямо в детализации;
указать одну себестоимость для всех баркодов артикула;
указать отдельную себестоимость для каждого баркода.
При указании даты изменения себестоимости необходимо помнить, что дата изменения себестоимости — это не дата поставки товара на склад маркетплейса и не дата, когда начались заказы этого товара. Дата, которую пользователь указывает при внесении себестоимости по периодам — это дата, начиная с которой сервис будет рассчитывать продажи (выкупы на ПВЗ) по новой себестоимости.
Соответственно, если у продавца на складе маркетплейса ещё есть товары этого же артикула со старым значением себестоимости, необходимо, используя коэффициент «Средняя скорость продаж» из раздела «Склад» для данного артикула, рассчитать, на сколько дней хватит текущего остатка. Либо можно воспользоваться уже предрассчитанным количеством дней остатка товара на складе со старой себестоимостью, которое также отображается в разделе «Склад».
Исходя из того, на сколько дней хватит остатка товара со старой себестоимостью, пользователь должен определить дату, когда этот остаток закончится, и дополнительно прибавить ещё 3–4 дня. Эти 3–4 дня закладываются как среднее время доставки товара — от момента оформления заказа покупателем в приложении до момента получения товара в пункте выдачи заказов.
Соответственно, от момента фактической поставки новой партии товара с новой себестоимостью на склад маркетплейса до даты, которую необходимо указать для корректного изменения себестоимости, должно пройти количество дней, рассчитанное следующим образом:
количество дней, необходимое для продажи предыдущей партии товара при текущей скорости продаж;
плюс дополнительные 3–4 дня на то, чтобы покупатели начали получать новую партию товара с новой себестоимостью в пунктах выдачи заказов.
Загрузка себестоимости через таблицу
Себестоимость в разделе «Товары» можно загрузить с помощью таблицы.
Для того чтобы скачать таблицу, необходимо нажать кнопку «Скачать таблицу», расположенную в левом нижнем углу страницы «Товары».
После нажатия на кнопку «Скачать таблицу» начнётся автоматическое скачивание шаблона в формате Excel.
Таблицу необходимо заполнить согласно инструкции, указанной в верхней части скачанного шаблона.
Проверка данных перед загрузкой
Для магазинов Wildberries необходимо проверить, что у товаров заполнены:
баркод;
артикул;
артикул продавца;
цена;
при необходимости — даты изменения цены.
Для товаров из магазинов Ozon необходимо проверить, что заполнены:
артикул;
артикул продавца;
цена;
при необходимости — даты изменения цены.
Баркоды для товаров Ozon отсутствуют и не предусмотрены.
Соответственно, перед загрузкой необходимо убедиться, что у всех товаров корректно заполнены обязательные поля, чтобы таблица была обработана системой без ошибок.
Внесение себестоимости по периодам через таблицу
В таблице можно внести себестоимость «за всё время» — в этом случае для каждого товара указывается только одно значение себестоимости.
Если необходимо внести себестоимость через таблицу по периодам, пользователь должен:
полностью скопировать строку с нужным товаром;
вставить новую строку непосредственно ниже;
либо скопировать сразу массив строк и добавить их ниже всей таблицы.
Правила заполнения таблицы
После завершения работы с таблицей необходимо убедиться, что:
для каждого товара есть одна или несколько строк;
если у товара две строки и более, то одна из строк должна содержать пустое значение в ячейке «Дата»;
ячейки «Цена» заполнены у всех товаров;
для товаров, у которых себестоимость изменялась с определённой даты, одна из строк содержит дату изменения цены;
формат ячеек таблицы имеет значение “Обычный текст”.
Загрузка таблицы в сервис
После заполнения таблица загружается в сервис с помощью кнопки «Загрузить таблицу», расположенной также в левом нижнем углу страницы «Товары».
При нажатии на кнопку «Загрузить таблицу» откроется окно выбора документа на локальном устройстве пользователя.
После выбора файла начинается:
загрузка таблицы;
оцифровка данных;
обработка информации о себестоимости.
В зависимости от объёма данных и количества строк в документе загрузка может занимать продолжительное время — иногда до получаса, редко - более.
Отчёт импорта
После завершения загрузки пользователю отображается отчёт импорта.
В отчёте указывается:
количество строк в загруженном документе;
количество успешно загруженных строк;
количество строк, по которым были выявлены ошибки или данные не удалось загрузить.
Ниже отображается детализация ошибок:
код ошибки;
наименование ошибки.
Каждую ошибку можно раскрыть с помощью аккордеона и посмотреть дочерние строки, где будут указаны:
номер строки документа;
артикул;
наименование товара.
Это позволяет быстро определить, по каким товарам возникли проблемы при загрузке.
Частые ошибки при загрузке
Самые частые ошибки, которые могут возникать при загрузке товаров, связаны:
с некорректным форматом самого документа;
с некорректным форматом ячеек в документе.
При загрузке необходимо убедиться, что все ячейки документа имеют формат:
либо «Обычный текст»;
либо стандартный формат ячеек Excel.
Также к стандартным ошибкам относится ситуация, когда пользователь указал для одного товара несколько строк, но:
оставил поле «Себестоимость» пустым;
либо оставил поле «Дата» пустым при наличии нескольких строк и одинакового значения себестоимости;
либо создал две строки одного и того же товара с одинаковой себестоимостью, но без указания даты;
либо указал значения даты для всех строк одного артикула и при этом не оставил ни одной строки с пустой ячейкой даты для данного товара.
То есть если для одного товара в таблице указано несколько строк, одна из строк обязательно должна содержать пустое значение в поле «Дата». Это необходимо для корректной обработки изменений себестоимости системой.Действия при ошибках
В таком случае необходимо:
Проверить заполнение файла.
Проверить строки, указанные в отчёте об ошибках.
Исправить заполнение проблемных строк.
Свериться с инструкцией, расположенной в верхней части шаблона для загрузки себестоимости товаров.
Для магазинов Озон доступен импорт себестоимости, заполненной в кабинете Озон.
